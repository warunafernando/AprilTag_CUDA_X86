CONCLUSION:
----------
The CPU detector correctly identifies ONE tag #3 per frame.
The CUDA detector produces 3-4 duplicate detections of the same tag per frame.

This indicates an issue with the CUDA detector's duplicate filtering logic.
The reconcile_detections() function in the CUDA code is not properly removing
duplicate detections of the same tag ID.

RECOMMENDATION:
--------------
Use the CPU detector videos (Stable_3d_cpu.avi, Moving_3d_cpu.avi) as the
reference for correct tag detection. The 3D visualization in these videos
should show the tag at the correct location.

The CUDA detector needs its duplicate filtering improved to match the CPU
detector's behavior.
